A. 视图最终收敛 (View Convergence)

针对 NFS 无 Agent 节点的“监控盲区”进行了暴力测试。在压力环境下随机在多个盲区节点执行海量文件操作。

实现原理: Fustor 采用 Scrutiny 差分算法。Leader 节点执行深度物理扫描，将物理磁盘现状与内存元数据进行异或对比，强制补全缺失的 VFS 事件，确保接口视图在审计周期后实现最终一致性。

B. 逻辑删除单调性 (Deletion Monotonicity)

模拟极其严重的 NFS acregmin 延迟。测试在节点 A 手动删除条目后，节点 B 仍上报陈旧目录项的极端情况。

实现原理: 引入分布式 Tombstone（墓碑）。一旦收到 Realtime Delete 事件，Fusion 会持久化一个带逻辑位的时间戳。所有后续来自 Scan 模式的数据必须经过物理 Mtime 与墓碑逻辑时间的对撞校验，只有严格更新的版本才允许“转世”。
coverage: test_d2, test_d3, test_d4

C. 静默态识别 (Quiescence Verification)

测试在高频 IO 写入流中，API 视图对“半截文件”的预警能力。确保用户不会在数据未 Commit 时读入下游。

实现原理: 热数据追踪。对于所有非实时上报的文件变更，系统自动降级信任度并放入 Suspect List。由后台 Sentinel 协程发起主动 `stat` 系统调用，通过采样探测文件属性是否已进入静默期，从而动态控制 `integrity_suspect` 标志。
coverage: test_c1, test_c6
